Why Storage Matters in Forensics

Before we chase malware through boot sectors or recover deleted partitions, we need to understand where data lives. Storage devices are the backbone of digital evidence. Whether it’s a bootkit hiding in sector 0 or a deleted file lingering in slack space, your investigation starts with knowing the medium.

Categories of Storage Devices

Primary Storage (Volatile)

  • RAM: Fast, temporary storage used by active processes. Volatile—data lost on shutdown.
  • Cache Memory: Ultra-fast, stores frequently accessed data. Rarely accessible in forensic workflows.

Secondary Storage (Non-Volatile)

  • HDD: Magnetic platters, mechanical read/write heads. Excellent for long-term data retention.
  • SSD: NAND flash memory. Fast, but TRIM and wear leveling complicate recovery.

Tertiary & Offsite Storage

  • Optical Discs: Legacy archival media.
  • Magnetic Tape: High-capacity, long-term backup.
  • Cloud Storage: Remote systems. Forensic access depends on API logs and sync clients.

Useful Commands

Windows

Get-PhysicalDisk
Get-Disk
Get-Volume

Linux

lsblk
sudo fdisk -l
df -h

Logs & Artifacts

  • Windows: Event Viewer → System & Storage Diagnostics
  • Linux: /var/log/syslog, /var/log/dmesg

Recommended Tools

  • FTK Imager – Disk imaging and preview
  • Autopsy / SleuthKit – File system and metadata analysis
  • WinHex / HxD – Sector-level inspection
  • dd / dc3dd – Raw disk acquisition (Linux)

Forensic Insight

Understanding storage types helps you:

  • Choose the right acquisition method (live vs dead)
  • Predict artifact volatility (RAM vs SSD)
  • Identify hidden volumes and slack space
  • Prepare for challenges like TRIM, wear leveling, and cloud sync gaps
